Nvim-treesitter provides superior syntax highlighting, code navigation, and text objects using tree-sitter parsers for accurate code understanding.

## Overview

Treesitter builds an abstract syntax tree (AST) of your code, enabling:
- Accurate syntax highlighting
- Smart code navigation
- Syntax-aware text objects
- Better code folding
- Incremental parsing for performance

## Configuration

```lua lua/leyland/plugins/treesitter.lua
return {
  "nvim-treesitter/nvim-treesitter",
  branch = "master",
  event = { "BufReadPre", "BufNewFile" },
  build = ":TSUpdate",
  config = function()
    require("nvim-treesitter").setup({
      ensure_installed = {
        "json",
        "javascript",
        "typescript",
        "tsx",
        "yaml",
        "html",
        "css",
        "prisma",
        "markdown",
        "markdown_inline",
        "svelte",
        "graphql",
        "bash",
        "lua",
        "vim",
        "dockerfile",
        "gitignore",
        "query",
        "vimdoc",
        "c",
      },
    })
  end,
}
```

## Language Registration

Zsh files are configured to use the bash parser:

```lua
vim.treesitter.language.register("bash", "zsh")
```

<Tip>
This means `.zsh` files get proper syntax highlighting using the bash treesitter parser.
</Tip>

## Automatic Updates

The build command ensures parsers stay up to date:

```lua
build = ":TSUpdate"
```

This automatically runs `:TSUpdate` when the plugin is installed or updated, keeping all parsers current.

## Lazy Loading

```lua
event = { "BufReadPre", "BufNewFile" }
```

Treesitter loads before reading or creating files, ensuring syntax highlighting is immediately available.

## Benefits Over Regex Highlighting

<AccordionGroup>
  <Accordion title="Context Awareness">
    Treesitter understands code structure. For example, it knows the difference between a function name and a variable, even if they look the same.
  </Accordion>
  
  <Accordion title="Consistency">
    Highlighting is consistent across the entire file because treesitter parses the whole syntax tree, not just individual lines.
  </Accordion>
  
  <Accordion title="Performance">
    Incremental parsing means only changed portions of the file are re-parsed, making it fast even for large files.
  </Accordion>
  
  <Accordion title="Multi-language Support">
    In files with embedded languages (like JSX with CSS-in-JS), treesitter correctly highlights each language.
  </Accordion>
</AccordionGroup>

</Note>

## Troubleshooting

### Parser Not Working

If syntax highlighting isn't working for a language:

1. Check if the parser is installed:
   ```vim
   :TSModuleInfo
   ```

2. Try reinstalling the parser:
   ```vim
   :TSInstall! <language>
   ```

3. Update all parsers:
   ```vim
   :TSUpdate
   ```

### Performance Issues

For very large files, you can disable treesitter:

```vim
:TSBufDisable highlight
```
