## Essential Keybindings

nvim-luffy uses `Space` as the leader key. Here are the most important keybindings to get you started.

<Note>
All keybindings shown use `Space` as the leader. In Neovim notation, this is written as `<leader>`.
</Note>

### File Navigation

<Steps>
  <Step title="Find Files with Telescope">
    The fastest way to navigate your project:

    | Key | Action | Description |
    |-----|--------|-------------|
    | `Space Space` | Find files | Fuzzy search all files in current directory |
    | `Space /` | Live grep | Search for text across entire project |
    | `Space fr` | Recent files | Browse recently opened files |
    | `Space fc` | Find word under cursor | Search for word under cursor |
    | `Space fk` | Find keymaps | Search all available keybindings |

    <CodeGroup>
    ```lua telescope.lua (lines 38-42)
    keymap.set("n", "<leader><leader>", "<cmd>Telescope find_files<cr>")
    keymap.set("n", "<leader>fr", "<cmd>Telescope oldfiles<cr>")
    keymap.set("n", "<leader>/", "<cmd>Telescope live_grep<cr>")
    keymap.set("n", "<leader>fc", "<cmd>Telescope grep_string<cr>")
    keymap.set("n", "<leader>fk", "<cmd>Telescope keymaps<cr>")
    ```
    </CodeGroup>

    <Tip>
    In Telescope, use `Ctrl+j` and `Ctrl+k` to move through results, `Ctrl+q` to send results to quickfix list.
    </Tip>
  </Step>

  <Step title="Quick Jumps with Harpoon">
    Mark your most important files for instant access:

    | Key | Action | Description |
    |-----|--------|-------------|
    | `Space H` | Mark file | Add current file to harpoon list |
    | `Space h` | Quick menu | Open harpoon menu to see all marked files |
    | `Space 1-9` | Jump to file | Instantly jump to marked file by number |

    **Workflow example:**
    1. Open your main component file → Press `Space H`
    2. Open the related test file → Press `Space H`
    3. Open the API route → Press `Space H`
    4. Now press `Space 1`, `Space 2`, `Space 3` to instantly jump between them

    <CodeGroup>
    ```lua harpoon.lua (lines 14-28)
    {
      "<leader>H",
      function()
        require("harpoon"):list():add()
      end,
      desc = "Harpoon File",
    },
    {
      "<leader>h",
      function()
        local harpoon = require("harpoon")
        harpoon.ui:toggle_quick_menu(harpoon:list())
      end,
      desc = "Harpoon Quick Menu",
    }
    ```
    </CodeGroup>
  </Step>
</Steps>

### Window Management

<Steps>
  <Step title="Split Windows">
    | Key | Action |
    |-----|--------|
    | `Space \|` | Split window vertically |
    | `Space sh` | Split window horizontally |
    | `Ctrl+w` + `hjkl` | Navigate between splits |

    <CodeGroup>
    ```lua keymaps.lua (lines 11-12)
    keymap.set("n", "<leader>|", "<C-w>v", { desc = "Split window vertically" })
    keymap.set("n", "<leader>sh", "<C-w>s", { desc = "Split window horizontally" })
    ```
    </CodeGroup>
  </Step>

  <Step title="Buffer Management">
    | Key | Action |
    |-----|--------|
    | `Space bd` | Close current buffer |
    | `Space ba` | Close all buffers |
  </Step>
</Steps>

### Code Editing

<Steps>
  <Step title="Auto-Completion">
    nvim-cmp provides intelligent completion as you type:

    | Key | Action |
    |-----|--------|
    | `Ctrl+Space` | Trigger completion manually |
    | `Ctrl+j` / `Ctrl+k` | Navigate suggestions |
    | `Tab` | Select next item or expand snippet |
    | `Enter` | Accept selected completion |
    | `Ctrl+e` | Close completion menu |
    | `Ctrl+b` / `Ctrl+f` | Scroll documentation |

    <CodeGroup>
    ```lua nvim-cmp.lua (lines 34-41)
    mapping = cmp.mapping.preset.insert({
      ["<C-k>"] = cmp.mapping.select_prev_item(),
      ["<C-j>"] = cmp.mapping.select_next_item(),
      ["<C-b>"] = cmp.mapping.scroll_docs(-4),
      ["<C-f>"] = cmp.mapping.scroll_docs(4),
      ["<C-Space>"] = cmp.mapping.complete(),
      ["<C-e>"] = cmp.mapping.abort(),
      ["<CR>"] = cmp.mapping.confirm({ select = false }),
    ```
    </CodeGroup>

    <Info>
    Completion sources include LSP suggestions, snippets, buffer text, and file paths.
    </Info>
  </Step>

  <Step title="Code Formatting">
    Code is automatically formatted on save using Conform.nvim.

    | Key | Action |
    |-----|--------|
    | `Space mp` | Format file or selection |
    | `:w` | Auto-format on save (automatic) |

    **Supported formatters:**
    - **JavaScript/TypeScript/JSON/CSS/HTML**: Prettier
    - **Lua**: Stylua
    - **Python**: Black + isort

    <CodeGroup>
    ```lua formatting.lua (lines 24-28)
    format_on_save = {
      lsp_fallback = true,
      async = false,
      timeout_ms = 3000,
    }
    ```
    </CodeGroup>

    <Tip>
    Formatting happens automatically on save with a 3-second timeout. If a formatter isn't found, LSP formatting is used as fallback.
    </Tip>
  </Step>

  <Step title="LSP Features">
    Language Server Protocol provides intelligent code navigation:

    | Key | Action |
    |-----|--------|
    | `K` | Show hover documentation |
    | `gd` | Go to definition |
    | `gr` | Find references |
    | `gi` | Go to implementation |
    | `Space ca` | Code actions |
    | `Space rn` | Rename symbol |

    <Note>
    These are standard LSP keybindings. Check `:LspInfo` to see which servers are active for the current buffer.
    </Note>
  </Step>
</Steps>

### Git Integration

<Steps>
  <Step title="Neogit - Git Interface">
    A powerful Magit-like interface for Git operations:

    | Key | Action |
    |-----|--------|
    | `Space gg` | Open Neogit in floating window |

    **Inside Neogit:**
    - `s` - Stage/unstage file or hunk
    - `c` - Commit (opens commit message buffer)
    - `P` - Push
    - `p` - Pull
    - `?` - Show help

    <CodeGroup>
    ```lua neogit.lua (lines 13-20)
    keys = {
      {
        "<leader>gg",
        function()
          require("neogit").open({ kind = "floating" })
        end,
        desc = "Neogit (floating)",
      },
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Diffview - Visual Diffs">
    Compare changes visually with Diffview:

    | Key | Action |
    |-----|--------|
    | `Space dd` | Open diff view |
    | `Space ds` | Compare current branch with origin/main |
    | `Space df` | Close diff view |

    <CodeGroup>
    ```lua diffview.lua (lines 4-8)
    keys = {
      { "<leader>dd", "<cmd>DiffviewOpen<cr>", desc = "Open diff view" },
      { "<leader>df", "<cmd>DiffviewClose<cr>", desc = "Close diff view" },
      { "<leader>ds", "<cmd>:DiffviewOpen origin/main...HEAD<cr>" },
    }
    ```
    </CodeGroup>
  </Step>
</Steps>

## Common Workflows

<AccordionGroup>
  <Accordion title="Starting a new feature">
    <Steps>
      <Step title="Create feature branch">
        ```bash
        git checkout -b feature/new-feature
        ```
      </Step>
      <Step title="Mark key files with Harpoon">
        As you identify files you'll frequently edit:
        - Open each file
        - Press `Space H` to mark it
        - Use `Space 1-9` to jump between them
      </Step>
      <Step title="Use Telescope for exploration">
        - `Space Space` to find files by name
        - `Space /` to search for specific code patterns
        - `Space fc` to find all usages of a function
      </Step>
      <Step title="Commit changes with Neogit">
        - `Space gg` to open Neogit
        - Stage files with `s`
        - Press `c` to commit
        - Write commit message and save
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Debugging LSP issues">
    <Steps>
      <Step title="Check LSP status">
        ```vim
        :LspInfo
        ```
        Shows active language servers for current buffer
      </Step>
      <Step title="View LSP logs">
        ```vim
        :LspLog
        ```
        See detailed logs if something isn't working
      </Step>
      <Step title="Restart LSP">
        ```vim
        :LspRestart
        ```
        Restart the language server for current buffer
      </Step>
      <Step title="Check Mason installations">
        ```vim
        :Mason
        ```
        Verify language servers are installed (✓ icon)
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Reviewing changes before commit">
    <Steps>
      <Step title="View all changes">
        ```vim
        Space dd
        ```
        Opens Diffview showing all modified files
      </Step>
      <Step title="Compare with main branch">
        ```vim
        Space ds
        ```
        Shows diff between your branch and origin/main
      </Step>
      <Step title="Stage specific hunks in Neogit">
        - `Space gg` to open Neogit
        - Navigate to a file and press `Tab` to expand
        - Use `s` on individual hunks to stage them selectively
      </Step>
      <Step title="Close diff view">
        ```vim
        Space df
        ```
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Working with multiple projects">
    <Steps>
      <Step title="Navigate to project directory">
        ```vim
        :cd ~/projects/my-app
        ```
        Change working directory to your project
      </Step>
      <Step title="Find files in new directory">
        `Space Space` now searches the new directory
      </Step>
      <Step title="Use sessions (optional)">
        Consider using a session plugin to save your window layout, buffers, and marks per-project
      </Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## Configuration Quick Tips

<CardGroup cols={2}>
  <Card title="Change colorscheme" icon="palette">
    Edit `lua/leyland/plugins/colorscheme.lua` and change:
    ```lua
    vim.cmd.colorscheme("your-theme")
    ```
  </Card>

  <Card title="Adjust editor settings" icon="gear">
    Modify `lua/leyland/core/options.lua` for settings like:
    - Tab width (tabstop, shiftwidth)
    - Line numbers (relativenumber)
    - Wrapping (wrap)
  </Card>

  <Card title="Add custom keymaps" icon="keyboard">
    Add to `lua/leyland/core/keymaps.lua`:
    ```lua
    keymap.set("n", "<leader>x", ":YourCommand<CR>")
    ```
  </Card>

  <Card title="Install more LSP servers" icon="code">
    Edit `lua/leyland/plugins/lsp/mason.lua` and add to `ensure_installed`
  </Card>
</CardGroup>
